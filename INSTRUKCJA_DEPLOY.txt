INSTRUKCJA: DEPLOY ZMIAN
========================
Folder projektu: C:\Users\filip.zarazinski\OneDrive - AddSecure\Desktop\aplikacja odbiory\files


== WAŻNE: Na początku każdej sesji PowerShell dodaj git do PATH ==

  $env:PATH = "C:\Program Files\Git\bin;" + $env:PATH
  cd "C:\Users\filip.zarazinski\OneDrive - AddSecure\Desktop\aplikacja odbiory\files"


== 1. WYSŁANIE ZMIAN NA GITHUB (bez przebudowy) ==

  git add -A
  git commit -m "opis zmian"
  git push


== 2. PRZEBUDOWA EXE ==

  Remove-Item -Recurse -Force dist, build   # wyczyść poprzedni build (jesli byl blad)
  pyinstaller Odbiory.spec

Plik EXE bedzie w: dist\Odbiory\Odbiory.exe


== 3. BUDOWA INSTALLERA (Inno Setup) ==

Otwórz Inno Setup Compiler → File → Open → wybierz installer.iss → kliknij Build (F9)

Installer bedzie w: installer_output\Odbiory_Setup.exe


== 4. WGRANIE NOWEGO RELEASE NA GITHUB ==

Przed release pamietaj o podbiciu wersji w:
  - config.py   → APP_VERSION = "X.X.X"
  - version.txt → X.X.X
  - installer.iss → AppVersion = "X.X.X"

Potem wyslij zmiany na gita (punkt 1), a nastepnie:

  & "C:\Program Files\GitHub CLI\gh.exe" release create vX.X.X "installer_output\Odbiory_Setup.exe" --title "Odbiory vX.X.X" --notes "- Opis zmian"

Przyklad dla wersji 1.0.3:

  & "C:\Program Files\GitHub CLI\gh.exe" release create v1.0.3 "installer_output\Odbiory_Setup.exe" --title "Odbiory v1.0.3" --notes "- Poprawka X"


== 5. PELNY FLOW OD ZMIANY DO RELEASE ==

  1. Wprowadz zmiany w kodzie
  2. Zaktualizuj APP_VERSION w config.py, version.txt i installer.iss
  3. git add -A && git commit -m "opis" && git push
  4. Remove-Item -Recurse -Force dist, build
  5. pyinstaller Odbiory.spec
  6. Inno Setup → F9
  7. & "C:\Program Files\GitHub CLI\gh.exe" release create vX.X.X ...


== UWAGI ==

- Przed uzyciem gh trzeba byc zalogowanym:
    & "C:\Program Files\GitHub CLI\gh.exe" auth login

- Skroty Inno Setup:
    F9      = kompiluj
    Ctrl+O  = otworz plik
